## Megjegyzések

- A létrehozott fájlok jelenleg `placeholder` szerepet töltenek be.
- A struktúra célja az induló boundary-k rögzítése.
- A tényleges implementációt később modulonként érdemes felépíteni.
- Az `analytics` modul induláskor lekérdezés-orientált maradhat.
- Az új dokumentumok a fejlesztési sorrendet és a migration stratégiát rögzítik.
- Az `IDENTITY_CORE_MODEL_PLAN.md` az első identity és core master data ORM modelltervet rögzíti.
- A `master_data` modul most külön backend modulként is megjelent.
- Az első `identity` és `master_data` ORM modellek már nem csak placeholderként, hanem SQLAlchemy skeletonként szerepelnek.
- A `master_data` modul első olvasási repository és query skeletonjai is bekerültek.
